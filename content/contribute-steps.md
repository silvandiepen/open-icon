# Contribute steps

We welcome contributions from designers who share our commitment to creating high-quality, consistent, and accessible icons. Here's how to make a pull request to add or edit icons in our library:

1. **Fork the Repository:** Start by forking our repository on [GitHub](https://github.com/OpenIconFoundation) to create a copy of it in your own GitHub account.

2. **Create a New Branch:** Create a new branch in your forked repository. Give it a descriptive name related to the icon(s) you plan to work on.

3. **Design Your Icons:** Using vector design software (e.g., Adobe Illustrator, Inkscape), create your icons following the guidelines mentioned above.

4. **File Format:** Save your icons in SVG format. Ensure that your SVG file contains only vector shapes and lines without any raster elements.

5. **Documentation:** Include a README file in your pull request that provides a brief description of the icon(s) and any relevant licensing information. Make it easy for reviewers to understand the purpose of your icons.

6. **Commit and Push:** Commit your changes to your branch and push them to your forked repository on GitHub.

7. **Submit a Pull Request:** Go to the original Open Icon Foundation repository and create a pull request from your branch. Describe your changes and icons in the pull request description.

8. **Review and Collaboration:** Our team will review your pull request for adherence to our guidelines and quality standards. We may provide feedback and collaborate with you to ensure your icons meet our criteria.

9. **Merge and Acknowledgment:** Once your pull request is approved, it will be merged into our main repository, and your contribution will be acknowledged in our list of contributors.

By following these guidelines and steps, you can contribute to our mission of providing a comprehensive library of high-quality, standardized icons that empower creativity and accessibility for everyone.
